## Architecture â€” Dynamic CRUD System

This is **not** a conventional page-per-feature app. It uses a **dynamic entity-driven CRUD** system:

- Backend exposes entity metadata via REST at `http://localhost/api/entity_configuration_dtos`
- Routes `/lista/:entity` and `/form/:entity/:id?` render `DynamicCollection.vue` and `DynamicForm.vue` components
- Stores are created **at runtime** by `storeFactory(src/stores/storeFactory.ts)` based on entity name
- Entity definitions (fields, queries, mutations) are registered via `entityRegistry.ts` and consumed from the GraphQL backend
- Per-entity route files (e.g., `src/router/user.ts`, `src/router/role.ts`) add static overrides on top of dynamic routes

**Key stores:**
- `storeFactory.ts` â€” primary, active factory for dynamic entity stores (collection, form, CRUD operations via GraphQL)
- `storeFactory copy.ts`, `storeFactory2.ts` â€” stale copies, ignore
- `stores/autoimport/` â€” auto-imported global stores (loading, menu, sidebar)

## Data Layer

- **GraphQL** (primary): Apollo Client 4 (`@apollo/client`) with custom link chain: mutation â†’ auth â†’ error â†’ loading â†’ removeTypename â†’ http. Default fetchPolicy is `no-cache`.
- **REST** (secondary): `useApiRest` composable with JWT token in localStorage, refresh via `/auth/refresh`, route-change cancellation.
- **Mercure**: Real-time subscriptions via `mercureItem.ts` / `mercureList.ts` composables.

Both layers point to `http://localhost/api` (REST) and `http://localhost/api/graphql` (GraphQL). Change in `src/config/config.ts`.
